# 博客日常操作

这份文档记录写文章、插入图片、插入公式、预览和发布的固定流程。

## 目录

- 文章目录：`src/content/posts`
- 图片目录：`public/images`
- 发布分支：`main`
- 线上地址：`https://Starshone-shiyin.github.io/Blog/`

## 新建文章

在 `src/content/posts` 里新建一个 `.md` 文件，例如：

```text
src/content/posts/my-note.md
```

文件开头必须包含 frontmatter：

```md
---
title: "文章标题"
description: "文章摘要，会显示在文章列表和 SEO 描述里。"
pubDate: 2026-06-11
tags: ["随笔", "技术"]
---
```

正文从 frontmatter 后面开始写：

```md
这里是正文。

## 小标题

继续写内容。
```

## 插入图片

推荐把图片放在 `public/images/posts/文章名/` 下面，例如：

```text
public/images/posts/my-note/example.jpg
```

Markdown 里使用线上路径：

```md
![图片说明](/Blog/images/posts/my-note/example.jpg)
```

注意：

- 路径必须带 `/Blog`，因为 GitHub Pages 站点部署在 `/Blog/` 子路径下。
- 图片文件名建议使用英文、数字和短横线，例如 `cover.jpg`、`step-01.png`。
- 不建议使用很大的原图。网页图片通常控制在 300 KB 到 1 MB 左右更合适。

## 插入公式

项目已经支持 KaTeX 数学公式。

行内公式：

```md
这是行内公式：$E = mc^2$。
```

块级公式：

```md
$$
\int_0^1 x^2 dx = \frac{1}{3}
$$
```

常见写法：

```md
$$
a^2 + b^2 = c^2
$$

$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

## 本地预览

第一次打开项目或依赖变化后执行：

```powershell
npm install
```

启动本地预览：

```powershell
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:4321/Blog/
```

## 发布到网站

写完文章或放好图片后执行：

```powershell
git status
git add .
git commit -m "Add new blog post"
git push
```

推送后 GitHub Actions 会自动构建并部署，不需要每次手动重新配置。

查看部署状态：

```text
https://github.com/Starshone-shiyin/Blog/actions
```

状态变成绿色后，线上网站会更新。GitHub Pages 有时会有几十秒到几分钟缓存延迟。

## 常见问题

图片线上不显示：

- 检查图片是否已经放进 `public/images`。
- 检查 Markdown 路径是否以 `/Blog/images/` 开头。
- 检查文件名大小写是否一致。
- 检查是否已经 `git add` 并 `git push`。

公式不显示：

- 确认使用 `$...$` 或 `$$...$$`。
- 如果公式里有特殊符号，优先用块级公式。
- 本地执行 `npm run build` 看是否有报错。

文章没有出现：

- 检查文章是否放在 `src/content/posts`。
- 检查 frontmatter 是否完整。
- 检查 `draft` 是否被设置成 `true`。如果有 `draft: true`，文章不会发布。

发布失败：

- 打开 GitHub Actions 查看失败日志。
- 本地先执行 `npm run build`，如果本地构建失败，线上也会失败。
